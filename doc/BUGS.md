## 🔴 Открытые баги

### BUG-4 — Курсор «палец» только в области Leaflet-интерактива
**Статус**: ❌ Не исправлен  
**Описание**: В режиме Select (V) курсор `pointer` появляется только в синем прямоугольнике (Leaflet interactive overlay), а не по всей карте.  
**Root cause**: Leaflet принудительно выставляет свой `cursor` на `.leaflet-container` и `.leaflet-interactive`. CSS-правило `#map { cursor: pointer }` перебивается Leaflet'ом.  
**Правильный fix**: Менять курсор через `map.getContainer().style.cursor = 'pointer'` в JS-функции `setTool()`, а не через CSS-класс на body. Для `.leaflet-interactive` дополнительно переопределить через CSS с `!important`.

### BUG-6 — Тень у текста осталась
**Статус**: ❌ Не исправлен  
**Описание**: Текстовые метки на карте отображаются с чёрной тенью.  
**Root cause**: `text-shadow` был убран из CSS-класса `.text-marker-wrap`, но функция `buildTextIcon` формирует HTML-строку с инлайн-стилем `style="font:...;color:..."` — и там `text-shadow` никогда и не было прописан явно. Реальный источник тени — **браузерный/Leaflet default** или унаследованный стиль. Нужно явно добавить `text-shadow: none` прямо в инлайн-style строки `buildTextIcon`.  
**Правильный fix**: В `buildTextIcon` добавить `text-shadow:none` в инлайн-стиль div.

---

### BUG-9 — Пробел не работает как временный pan
**Статус**: ❌ Не исправлен  
**Описание**: Зажатие Space не переключает в режим перетаскивания карты. В логе нет никаких записей о нажатии.  
**Root cause**: `QWebEngineView` наследует от `QAbstractScrollArea`, которая **перехватывает Space** на уровне Qt до того, как событие достигает JS. JS-слушатель `document.addEventListener('keydown')` никогда не получает это событие.  
**Правильный fix**: Реализовать через Python: добавить `eventFilter` на `QWebEngineView` (или его viewport), перехватывать `QEvent.Type.KeyPress` с `Qt.Key.Key_Space`, и при нажатии вызывать `_js("startSpacePan()")`, при отпускании — `_js("stopSpacePan()")`.

### BUG-LOG — Лог нельзя выделить и скопировать
**Статус**: ❌ Не исправлен  
**Описание**: Текст в debug-панели (Ctrl+`) не выделяется мышью.  
**Root cause**: `user-select: text` в CSS не работает в `QWebEngineView` без явного разрешения. Также `pointer-events: auto` недостаточно — нужен `QWebEngineSettings.WebAttribute.JavascriptCanAccessClipboard = True` и/или кнопка «Копировать всё» внутри лога.  
**Правильный fix**: 
1. В `main.py` добавить `settings.setAttribute(QWebEngineSettings.WebAttribute.JavascriptCanAccessClipboard, True)`
2. В `#debug-log` добавить кнопку «📋 Копировать» которая вызывает `navigator.clipboard.writeText(debugEl.innerText)`.

### BUG-1 — При запуске не открывается последний проект и не полный экран
**Статус**: ⏳ Не начат  
**Описание**: При старте приложения не открывается последний открытый проект. Окно не разворачивается на весь экран автоматически.  
**Fix план**: 
- Сохранять `last_project` в `config.json` при каждом открытии/сохранении.
- При старте: `win.showMaximized()` вместо `win.show()`.
- После загрузки карты (`_on_map_loaded`) — если `last_project` существует, вызвать `_open_project(path)`.

---

### BUG-2 — Созданный объект не отображается в панели слоёв
**Статус**: ⏳ Не начат  
**Описание**: После рисования линии или размещения текста объект добавляется в `layers[id].objects` в JS, но счётчик/список объектов в Qt-панели слоёв не обновляется.  
**Fix план**: После `finishDrawing()` и `placeText()` вызывать `sendLayersToQt()` для синхронизации. В Qt-панели отображать кол-во объектов рядом с именем слоя.

### BUG-3 — Слой нельзя переименовать (ЧАСТИЧНО ИСПРАВЛЕН)
**Статус**: ✅ Исправлен в коммите `1f74cbd`  
**Описание**: Двойной клик перехватывался встроенным inline-edit QTreeWidget.  
**Fix**: `setEditTriggers(NoEditTriggers)` — теперь двойной клик открывает QInputDialog.

### BUG-7 — Двойной клик на текст: смена стиля
**Статус**: ⏳ Частично (мост настроен, обработчик в Python не добавлен)  
**Описание**: JS отправляет `bridge.onStatus('__EDIT_TEXT__:{json}')` при двойном клике, но в `main.py` в `on_js_status` нет обработки этого сообщения.  
**Fix план**: В `on_js_status` распарсить `__EDIT_TEXT__:`, показать `QDialog` с полями текст/размер/цвет/жирность/наклон, после `accept` вызвать `_js(f'applyTextEdit(...)')`.

---

### BUG-8 — Ctrl+Z двойное срабатывание (ИСПРАВЛЕН)
**Статус**: ✅ Исправлен в коммите `1f74cbd`  
**Описание**: `Ctrl+Z` обрабатывался и в Python (QShortcut), и в JS (keydown) — двойное срабатывание.  
**Fix**: Убран `Ctrl+Z` из JS keydown-слушателя.
