## 🔴 Открытые баги

### BUG-4 — Курсор «палец» только в области Leaflet-интерактива
**Статус**: ❌ Не исправлен  
**Описание**: В режиме Select (V) курсор `pointer` появляется только в синем прямоугольнике (Leaflet interactive overlay), а не по всей карте.  
**Root cause**: Leaflet принудительно выставляет свой `cursor` на `.leaflet-container` и `.leaflet-interactive`. CSS-правило `#map { cursor: pointer }` перебивается Leaflet'ом.  
**Правильный fix**: Менять курсор через `map.getContainer().style.cursor = 'pointer'` в JS-функции `setTool()`, а не через CSS-класс на body. Для `.leaflet-interactive` дополнительно переопределить через CSS с `!important`.

### BUG-9 — Пробел не работает как временный pan
**Статус**: ❌ Не исправлен  
**Описание**: Зажатие Space не переключает в режим перетаскивания карты. В логе нет никаких записей о нажатии.  
**Root cause**: `QWebEngineView` наследует от `QAbstractScrollArea`, которая **перехватывает Space** на уровне Qt до того, как событие достигает JS. JS-слушатель `document.addEventListener('keydown')` никогда не получает это событие.  
**Правильный fix**: Реализовать через Python: добавить `eventFilter` на `QWebEngineView` (или его viewport), перехватывать `QEvent.Type.KeyPress` с `Qt.Key.Key_Space`, и при нажатии вызывать `_js("startSpacePan()")`, при отпускании — `_js("stopSpacePan()")`.

### BUG-1 — При запуске не открывается последний проект и не полный экран
**Статус**: ⏳ Не начат  
**Описание**: При старте приложения не открывается последний открытый проект. Окно не разворачивается на весь экран автоматически.  
**Fix план**: 
- Сохранять `last_project` в `config.json` при каждом открытии/сохранении.
- При старте: `win.showMaximized()` вместо `win.show()`.
- После загрузки карты (`_on_map_loaded`) — если `last_project` существует, вызвать `_open_project(path)`.
