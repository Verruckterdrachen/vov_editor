## 🔴 Открытые баги / задачи


### BUG-14 — Текст не масштабируется при изменении зума карты
**Статус**: ❌ Не исправлен  
**Описание**: Размер текстовых меток (`L.divIcon`) фиксирован в пикселях и не меняется
при зуме карты, хотя линии (polyline) визуально масштабируются вместе с картой.  
**Root cause**: `divIcon` рендерится в CSS-пикселях экрана, не привязан к масштабу карты.  
**Механизм фикса**: Слушать `map.on('zoomend')`, пересчитывать и обновлять иконки всех
текстовых объектов: `scaledSize = obj.fontSize * 2^(zoom - baseZoom)`, clamp в разумных пределах.

### TASK-12 — Множественное выделение объектов
**Статус**: ❌ Не реализован  
**Описание**: Нет возможности выделить несколько объектов сразу для групповых операций (удаление и др.).  
**Минимум**: Shift+клик по объектам в панели слоёв — выделяет диапазон, Delete удаляет все выбранные.  
**Максимум**: Рисование области выделения на карте (drag в режиме select) — все объекты внутри выделяются.  
**Механизм (минимум)**: `selectedObjIds = []`, Shift+клик добавляет/убирает id. `deleteSelected` итерирует по всем id.  
**Механизм (максимум)**: `L.rectangle` при drag в режиме select, проверка `map.getBounds().contains(latlng)` для каждого объекта.

---

### TASK-10 — Структура слоёв как в Photoshop
**Статус**: ❌ Не реализован (откладываем на последнее)  
**Описание**: Линии суммируются внутри слоя невидимо (не показываются как отдельные объекты).
Текстовые объекты — показываются как листья. Структура: 📁 Папка → 📄 Слой → [T Текст].  
**Механизм**: В sendLayersToQt фильтровать объекты — слать только type === 'text'
и type === 'image' как листья. polyline — не слать.
